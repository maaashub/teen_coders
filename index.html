Day 2: Building Blocks of the Web!

What is HTML?

HTML stands for HyperText Markup Language.

  • It's NOT a programming language, but a markup language.
  • It uses "tags" to define the content and structure of a web page.
  • Think of it as the blueprint or skeleton of your website.
  • Every webpage you visit is built with HTML!

HTML Tags: The Building Blocks

Opening and Closing!

HTML uses "tags" to mark up elements. Most tags come in pairs:

  • An opening tag: <tagname>
  • A closing tag: </tagname>
  • The content goes between the tags.

Example:

<p>This is a paragraph.</p>

Some tags are "self-closing" or "void tags" (they don't need a closing tag), like <br> for a line break.

Basic HTML Document Structure

The Essential Blueprint

Every HTML page starts with a basic structure:


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>
  • <!DOCTYPE html>: Declares the document type.
  • <html>: The root element of an HTML page.
  • <head>: Contains meta-information (not visible on the page).
  • <body>: Contains all the visible content of the page.

Common HTML Elements: Headings & Paragraphs

Organizing Your Text

  • Headings (<h1> to <h6>): Define titles and subtitles. <h1> is most important.
    • <h1>Main Title</h1><h2>Subtitle</h2>
  • Paragraphs (<p>): For blocks of text.
    • <p>This is a sentence.</p>
  • Line Breaks (<br>): Inserts a single line break. (Self-closing).
    • Line 1.<br>Line 2.

Day 3: Organizing Content with Lists!

Unordered and Ordered Lists

Lists are super useful for organizing information:

  • Unordered Lists (<ul>): For items where order doesn't matter (e.g., shopping list). Each item is a <li>.
    • <ul><li>Item 1</li><li>Item 2</li></ul>
  • Ordered Lists (<ol>): For items where order matters (e.g., recipe steps). Each item is also a <li>.
    • <ol><li>Step 1</li><li>Step 2</li></ol>

Day 3: Semantic HTML

Giving Meaning to Your Code

Semantic HTML tags describe the meaning of content, not just how it looks. This helps browsers, search engines, and screen readers.

  • <header>: Introduction/navigation.
  • <nav>: Navigation links.
  • <main>: Dominant content.
  • <article>: Independent content (e.g., blog post).
  • <section>: Thematic grouping.
  • <aside>: Related but separate content (e.g., sidebar).
  • <footer>: Copyright, contact info.

Why? Improves accessibility and SEO!

Day 4: Connecting the Web with Links!

The Power of the Anchor Tag

Links are what make the "web" truly a web! We use the anchor tag: <a>

  • The href attribute specifies the destination URL.
  • The text between the tags is what the user clicks.
<a href="https://www.google.com">Go to Google</a>

Day 4: Types of Links

External, Internal, and Target Attributes

  • External Links: To another website (e.g., https://www.wikipedia.org).
  • Internal Links: To another page within your own website (e.g., about-me.html).
  • Target Attribute: Controls where the link opens:
    • _self (default): Same window/tab.
    • _blank: New window/tab (recommended for external links!).
  • Email Links: <a href="mailto:your.email@example.com">Email Me</a>

Day 5: Bringing Life to Your Page with Images!

The <img> Tag

The <img> tag embeds images. It's a self-closing tag.

  • src (source): Path to the image file.
  • alt (alternative text): Text description for accessibility and if the image fails to load.
<img src="my-picture.jpg" alt="A photo of me smiling">

Day 5: Image Paths and Sizing

Relative vs. Absolute Paths & Dimensions

  • Relative Paths: For images within your project (e.g., `images/my-pic.png`).
  • Absolute Paths (URLs): For images on other websites (e.g., `https://www.example.com/logo.png`).
  • Width and Height Attributes: Can set size in HTML (`width="300"`), but CSS is generally better for responsive design.

Day 6: Organizing Data with Tables!

Rows, Columns, and Cells

Tables display tabular data (rows and columns):

  • <table>: Container for the table.
  • <tr>: Table Row.
  • <th>: Table Header (bold, centered).
  • <td>: Table Data cell.

<table>
    <tr><th>Header</th><td>Data</td></tr>
</table>

Day 6: Table Enhancements

Captions, Headers, Bodies, and Footers

For better structure and accessibility:

  • <caption>: Table title.
  • <thead>, <tbody>, <tfoot>: Group table sections.
  • colspan: Cell spans multiple columns.
  • rowspan: Cell spans multiple rows.

<table>
    <caption>My Table</caption>
    <thead><tr><th>Col 1</th></tr></thead>
    <tbody><tr><td>Data</td></tr></tbody>
    <tfoot><tr><td colspan="2">Summary</td></tr></tfoot>
</table>

Day 7: Getting User Input with Forms! (Part 1)

The <form> and <input> Tags

Forms are essential for collecting information from users (e.g., login, contact forms, surveys).

  • <form>: The container for all form elements.
  • <label>: Provides a caption for an input element. Crucial for accessibility! Use the `for` attribute to link it to an input's `id`.
  • <input>: The most common form element. Its behavior changes based on the `type` attribute.

<form>
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" placeholder="Enter your username">
    <button type="submit">Submit</button>
</form>

Day 7: Common Input Types

Text, Password, Email, Number

  • type="text": Single-line text input.
  • type="password": Text input where characters are masked (e.g., dots or asterisks).
  • type="email": For email addresses. Browsers can validate format.
  • type="number": For numerical input. Often shows up/down arrows.
  • placeholder attribute: Provides a hint to the user about what to enter.
  • name attribute: Important for identifying the data when the form is submitted.

<label for="user_email">Email:</label>
<input type="email" id="user_email" name="user_email" placeholder="you@example.com">

Day 8: More Form Elements! (Part 2)

Checkboxes, Radios, Textareas, and Dropdowns

  • type="checkbox": Allows users to select zero or more options.
  • type="radio": Allows users to select exactly one option from a group (use the same `name` attribute for a group).
  • <textarea>: For multi-line text input (e.g., comments). Has `rows` and `cols` attributes.
  • <select> and <option>: Creates a dropdown list.
  • required attribute: Makes an input field mandatory before form submission.

<input type="checkbox" id="html" name="skill" value="html">
<label for="html">HTML</label>

<input type="radio" id="male" name="gender" value="male">
<label for="male">Male</label>

<textarea id="comments" name="comments" rows="4" cols="50"></textarea>

<select id="country" name="country">
    <option value="usa">USA</option>
    <option value="can">Canada</option>
</select>

Day 8: Date, Color, and Range Inputs

Specialized Inputs for Specific Data

  • type="date": For selecting a date. Often provides a calendar picker.
  • type="color": For selecting a color. Provides a color picker.
  • type="range": For selecting a value from a range (e.g., a slider). Use `min`, `max`, `value`, and `step` attributes.

<label for="dob">Date of Birth:</label>
<input type="date" id="dob" name="dob">

<label for="favcolor">Favorite Color:</label>
<input type="color" id="favcolor" name="favcolor" value="#63b3ed">

<label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100" value="50">

Day 9: Bringing Media to Your Page!

Audio and Video Tags

HTML allows you to embed audio and video directly into your webpages!

  • <audio>: Embeds sound content.
  • <video>: Embeds video content.
  • Both tags use the `src` attribute for the media file path.
  • `controls` attribute: Adds default play/pause, volume, etc.
  • `autoplay`: Starts playing automatically (use with caution!).
  • `loop`: Repeats the media when it ends.

<audio src="mysong.mp3" controls loop>
    Your browser does not support the audio element.
</audio>

<video src="myvideo.mp4" controls width="400" height="225" poster="video-thumbnail.jpg">
    Your browser does not support the video tag.
</video>

The text inside the tags is fallback content for browsers that don't support the media.

Day 9: Multiple Media Sources & YouTube Embeds

Ensuring Compatibility & External Content

  • <source> tag: Inside `audio` or `video`, allows you to provide multiple file formats for broader browser compatibility. The browser will use the first one it supports.
    • 
<video controls>
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    Your browser does not support the video tag.
</video>
  • Embedding YouTube/Vimeo: For popular video platforms, it's common to use their provided `<iframe>` embed code.
    • 
<iframe width="560" height="315" src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
    frameborder="0" allowfullscreen></iframe>
