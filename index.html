Day 2: Building Blocks of the Web!

What is HTML?

HTML stands for HyperText Markup Language.

  • It's NOT a programming language, but a markup language.
  • It uses "tags" to define the content and structure of a web page.
  • Think of it as the blueprint or skeleton of your website.
  • Every webpage you visit is built with HTML!

HTML Tags: The Building Blocks

Opening and Closing!

HTML uses "tags" to mark up elements. Most tags come in pairs:

  • An opening tag: <tagname>
  • A closing tag: </tagname>
  • The content goes between the tags.

Example:

<p>This is a paragraph.</p>

Some tags are "self-closing" or "void tags" (they don't need a closing tag), like <br> for a line break.

Basic HTML Document Structure

The Essential Blueprint

Every HTML page starts with a basic structure:


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>
  • <!DOCTYPE html>: Declares the document type.
  • <html>: The root element of an HTML page.
  • <head>: Contains meta-information (not visible on the page).
  • <body>: Contains all the visible content of the page.

Common HTML Elements: Headings & Paragraphs

Organizing Your Text

  • Headings (<h1> to <h6>): Define titles and subtitles. <h1> is most important.
    • <h1>Main Title</h1><h2>Subtitle</h2>
  • Paragraphs (<p>): For blocks of text.
    • <p>This is a sentence.</p>
  • Line Breaks (<br>): Inserts a single line break. (Self-closing).
    • Line 1.<br>Line 2.

Day 3: Organizing Content with Lists!

Unordered and Ordered Lists

Lists are super useful for organizing information:

  • Unordered Lists (<ul>): For items where order doesn't matter (e.g., shopping list). Each item is a <li>.
    • <ul><li>Item 1</li><li>Item 2</li></ul>
  • Ordered Lists (<ol>): For items where order matters (e.g., recipe steps). Each item is also a <li>.
    • <ol><li>Step 1</li><li>Step 2</li></ol>

Day 3: Semantic HTML

Giving Meaning to Your Code

Semantic HTML tags describe the meaning of content, not just how it looks. This helps browsers, search engines, and screen readers.

  • <header>: Introduction/navigation.
  • <nav>: Navigation links.
  • <main>: Dominant content.
  • <article>: Independent content (e.g., blog post).
  • <section>: Thematic grouping.
  • <aside>: Related but separate content (e.g., sidebar).
  • <footer>: Copyright, contact info.

Why? Improves accessibility and SEO!

Day 4: Connecting the Web with Links!

The Power of the Anchor Tag

Links are what make the "web" truly a web! We use the anchor tag: <a>

  • The href attribute specifies the destination URL.
  • The text between the tags is what the user clicks.
<a href="https://www.google.com">Go to Google</a>

Day 4: Types of Links

External, Internal, and Target Attributes

  • External Links: To another website (e.g., https://www.wikipedia.org).
  • Internal Links: To another page within your own website (e.g., about-me.html).
  • Target Attribute: Controls where the link opens:
    • _self (default): Same window/tab.
    • _blank: New window/tab (recommended for external links!).
  • Email Links: <a href="mailto:your.email@example.com">Email Me</a>

Day 5: Bringing Life to Your Page with Images!

The <img> Tag

The <img> tag embeds images. It's a self-closing tag.

  • src (source): Path to the image file.
  • alt (alternative text): Text description for accessibility and if the image fails to load.
<img src="my-picture.jpg" alt="A photo of me smiling">

Day 5: Image Paths and Sizing

Relative vs. Absolute Paths & Dimensions

  • Relative Paths: For images within your project (e.g., `images/my-pic.png`).
  • Absolute Paths (URLs): For images on other websites (e.g., `https://www.example.com/logo.png`).
  • Width and Height Attributes: Can set size in HTML (`width="300"`), but CSS is generally better for responsive design.

Day 6: Organizing Data with Tables!

Rows, Columns, and Cells

Tables display tabular data (rows and columns):

  • <table>: Container for the table.
  • <tr>: Table Row.
  • <th>: Table Header (bold, centered).
  • <td>: Table Data cell.

<table>
    <tr><th>Header</th><td>Data</td></tr>
</table>

Day 6: Table Enhancements

Captions, Headers, Bodies, and Footers

For better structure and accessibility:

  • <caption>: Table title.
  • <thead>, <tbody>, <tfoot>: Group table sections.
  • colspan: Cell spans multiple columns.
  • rowspan: Cell spans multiple rows.

<table>
    <caption>My Table</caption>
    <thead><tr><th>Col 1</th></tr></thead>
    <tbody><tr><td>Data</td></tr></tbody>
    <tfoot><tr><td colspan="2">Summary</td></tr></tfoot>
</table>

Day 7: Getting User Input with Forms! (Part 1)

The <form> and <input> Tags

Forms are essential for collecting information from users (e.g., login, contact forms, surveys).

  • <form>: The container for all form elements.
  • <label>: Provides a caption for an input element. Crucial for accessibility! Use the `for` attribute to link it to an input's `id`.
  • <input>: The most common form element. Its behavior changes based on the `type` attribute.

<form>
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" placeholder="Enter your username">
    <button type="submit">Submit</button>
</form>

Day 7: Common Input Types

Text, Password, Email, Number

  • type="text": Single-line text input.
  • type="password": Text input where characters are masked (e.g., dots or asterisks).
  • type="email": For email addresses. Browsers can validate format.
  • type="number": For numerical input. Often shows up/down arrows.
  • placeholder attribute: Provides a hint to the user about what to enter.
  • name attribute: Important for identifying the data when the form is submitted.

<label for="user_email">Email:</label>
<input type="email" id="user_email" name="user_email" placeholder="you@example.com">

Day 8: More Form Elements! (Part 2)

Checkboxes, Radios, Textareas, and Dropdowns

  • type="checkbox": Allows users to select zero or more options.
  • type="radio": Allows users to select exactly one option from a group (use the same `name` attribute for a group).
  • <textarea>: For multi-line text input (e.g., comments). Has `rows` and `cols` attributes.
  • <select> and <option>: Creates a dropdown list.
  • required attribute: Makes an input field mandatory before form submission.

<input type="checkbox" id="html" name="skill" value="html">
<label for="html">HTML</label>

<input type="radio" id="male" name="gender" value="male">
<label for="male">Male</label>

<textarea id="comments" name="comments" rows="4" cols="50"></textarea>

<select id="country" name="country">
    <option value="usa">USA</option>
    <option value="can">Canada</option>
</select>

Day 8: Date, Color, and Range Inputs

Specialized Inputs for Specific Data

  • type="date": For selecting a date. Often provides a calendar picker.
  • type="color": For selecting a color. Provides a color picker.
  • type="range": For selecting a value from a range (e.g., a slider). Use `min`, `max`, `value`, and `step` attributes.

<label for="dob">Date of Birth:</label>
<input type="date" id="dob" name="dob">

<label for="favcolor">Favorite Color:</label>
<input type="color" id="favcolor" name="favcolor" value="#63b3ed">

<label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100" value="50">

Day 9: Bringing Media to Your Page!

Audio and Video Tags

HTML allows you to embed audio and video directly into your webpages!

  • <audio>: Embeds sound content.
  • <video>: Embeds video content.
  • Both tags use the `src` attribute for the media file path.
  • `controls` attribute: Adds default play/pause, volume, etc.
  • `autoplay`: Starts playing automatically (use with caution!).
  • `loop`: Repeats the media when it ends.

<audio src="mysong.mp3" controls loop>
    Your browser does not support the audio element.
</audio>

<video src="myvideo.mp4" controls width="400" height="225" poster="video-thumbnail.jpg">
    Your browser does not support the video tag.
</video>

The text inside the tags is fallback content for browsers that don't support the media.

Day 9: Multiple Media Sources & YouTube Embeds

Ensuring Compatibility & External Content

  • <source> tag: Inside `audio` or `video`, allows you to provide multiple file formats for broader browser compatibility. The browser will use the first one it supports.
    • 
<video controls>
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    Your browser does not support the video tag.
</video>
  • Embedding YouTube/Vimeo: For popular video platforms, it's common to use their provided `<iframe>` embed code.
    • 
<iframe width="560" height="315" src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
    frameborder="0" allowfullscreen></iframe>

Day 11: Making it Pretty with CSS! (Part 1)

Introduction to CSS: Styling Your Webpages

Welcome to CSS! If HTML is the structure, CSS (Cascading Style Sheets) is the designer of your webpage. It controls how your HTML elements look.

  • What colors should your text be?
  • How big should your headings be?
  • Where should elements be placed on the page?
  • What fonts should you use?

CSS answers all these questions and more!

Day 11: CSS Syntax

Selectors, Properties, and Values

A basic CSS rule has three main parts:


selector {
    property: value;
    property: value;
}
  • Selector: Points to the HTML element(s) you want to style (e.g., `p`, `h1`, `body`).
  • Declaration: The `property: value;` pair.
    • Property: The specific style attribute you want to change (e.g., `color`, `font-size`, `background-color`).
    • Value: The setting for that property (e.g., `blue`, `16px`, `#333`).

Example: `p { color: blue; font-size: 16px; }`

Day 11: Ways to Add CSS - Inline Styles

Directly on HTML Elements (Not Recommended for Most Cases!)

Inline styles are added directly to an HTML element using the `style` attribute.


<h1 style="color: blue; text-align: center;">Hello World!</h1>
<p style="font-size: 18px; background-color: lightgray;">This paragraph has inline styles.</p>

Pros: Quick for small, unique changes.
Cons: Hard to manage for large sites, mixes content and style, not reusable.

Day 11: Ways to Add CSS - Internal Styles

Inside the <head> of Your HTML Document

Internal styles are placed inside a <style> tag within the <head> section of your HTML document.


<head>
    <style>
        body {
            background-color: #1a202c;
            color: #e2e8f0;
        }
        h1 {
            color: #63b3ed;
            text-align: center;
        }
        p {
            font-family: sans-serif;
            font-size: 1.1rem;
        }
    </style>
</head>

Pros: Styles are in one place for that page.
Cons: Not reusable across multiple pages, can make HTML files very long.

The Best Way to Style: External Style Sheets

For larger websites with many pages, the best way to add CSS is using an external stylesheet. This keeps your HTML clean and allows you to apply the same styles across multiple pages easily.

Day 12: CSS Selectors (Part 1)

Element, Class, and ID Selectors

Selectors are how CSS "finds" the HTML elements it needs to style.

  • Element Selector: Selects all instances of an HTML element. 
    p { color: green; } /* Styles all <p> tags */
  • Class Selector: Selects all elements with a specific `class` attribute. Use `.` followed by the class name. An element can have multiple classes. 
    
<p class="highlight">Important text</p>
.highlight { background-color: yellow; }
  • ID Selector: Selects a single, unique element with a specific `id` attribute. Use `#` followed by the ID name. An ID must be unique per page! 
    
<div id="main-header">Website Title</div>
#main-header { font-size: 3em; }

Remember: IDs are for unique elements, classes are for reusable styles!

Day 14: The CSS Box Model!

Content, Padding, Border, Margin

Every HTML element is treated as a rectangular box by the browser. Understanding the Box Model is crucial for layout and spacing.

  • Content: The actual content (text, image, video). Controlled by `width` and `height`.
  • Padding: Space between the content and the border. Pushes content away from the border.
  • Border: A line that goes around the padding and content.
  • Margin: Space outside the border. Pushes other elements away from this element.

div {
    width: 200px;
    height: 100px;
    padding: 20px; /* Space inside */
    border: 5px solid red; /* Line around */
    margin: 30px; /* Space outside */
}

Day 14: Padding, Border, Margin - Shorthands

Controlling Each Side & Collapsing Margins

  • Individual Sides: `padding-top`, `padding-right`, `padding-bottom`, `padding-left` (same for `margin`, `border`).
  • Shorthand (1 value): Applies to all 4 sides. 
    padding: 20px; /* 20px on all sides */
  • Shorthand (2 values): Top/Bottom, Left/Right. 
    padding: 10px 20px; /* 10px top/bottom, 20px left/right */
  • Shorthand (3 values): Top, Left/Right, Bottom. 
    padding: 10px 20px 30px; /* 10px top, 20px left/right, 30px bottom */
  • Shorthand (4 values): Top, Right, Bottom, Left (clockwise from top). 
    padding: 10px 20px 30px 40px;
  • Border Shorthand: `border: [width] [style] [color];` (e.g., `border: 2px solid #63b3ed;`)
  • Margin Collapsing: Vertical margins between block elements sometimes collapse into a single margin.

Day 15: Controlling Element Flow with `display`!

Block, Inline, and Inline-Block

The `display` property is fundamental to how elements behave on a page. Every HTML element has a default `display` value.

  • `display: block;`
    • Takes up the full width available.
    • Starts on a new line.
    • Can set `width`, `height`, `padding`, `margin`.
    • Examples: `<p>`, `<div>`, `<h1>`, `<li>`
  • `display: inline;`
    • Takes up only as much width as its content.
    • Does NOT start on a new line.
    • Cannot set `width` or `height`. `padding-top`/`bottom` and `margin-top`/`bottom` have no effect.
    • Examples: `<a>`, `<span>`, `<img>` (by default, but often styled as inline-block)
  • `display: inline-block;`
    • Behaves like an inline element (doesn't start new line).
    • But, you CAN set `width`, `height`, `padding`, and `margin`.
    • Useful for arranging elements side-by-side while giving them dimensions.

Day 16: Powerful Layouts with Flexbox! (Part 1)

Introduction to Flexbox: One-Dimensional Layout

Flexbox is a CSS layout module that makes it incredibly easy to arrange items in a single row or column. It's perfect for navigation bars, simple grids, and aligning content.

Flexbox works with two main concepts:

  • Flex Container: The parent element that holds the flex items. You apply `display: flex;` to this.
  • Flex Items: The direct children of the flex container.

<div class="container">
    <div class="item">Item 1</div>
    <div class="item">Item 2</div>
    <div class="item">Item 3</div>
</div>

/* CSS */
.container {
    display: flex; /* Makes this a flex container */
}

Day 16: Flex Container Properties

Direction and Alignment

  • `display: flex;` or `display: inline-flex;` (makes the container itself inline)
  • `flex-direction`: Defines the direction of the main axis (how items are laid out).
    • `row` (default): Items arranged horizontally.
    • `column`: Items arranged vertically.
    • `row-reverse`, `column-reverse`: Reverses the order.
  • `justify-content`: Aligns items along themain axis.
    • `flex-start` (default), `flex-end`, `center`, `space-between`, `space-around`, `space-evenly`
  • `align-items`: Aligns items along the cross axis (perpendicular to main axis).
    • `stretch` (default), `flex-start`, `flex-end`, `center`, `baseline`

Day 17: Flexbox Item Properties! (Part 2)

Controlling Individual Flex Items

While container properties control the overall layout, flex item properties allow you to control individual children:

  • `flex-grow`: Defines the ability of a flex item to grow if necessary. Takes a unitless value (e.g., `1`, `2`). Items with higher `flex-grow` values take up more available space.
  • `flex-shrink`: Defines the ability of a flex item to shrink if necessary.
  • `flex-basis`: Defines the default size of an element before the remaining space is distributed. Can be a length (e.g., `100px`) or a percentage (`50%`).
  • `flex` shorthand: Combines `flex-grow`, `flex-shrink`, and `flex-basis` (e.g., `flex: 1 1 0;`).
  • `align-self`: Overrides the container's `align-items` for a single flex item.
  • `order`: Controls the order in which flex items appear in the flex container. Defaults to `0`.

.item-grow { flex-grow: 1; }
.item-specific-align { align-self: flex-end; }
.item-first { order: -1; } /* Appears first */

Day 17: Responsive Considerations with Flexbox

Wrapping and Responsiveness

  • `flex-wrap`: Controls whether flex items are forced onto one line or can wrap onto multiple lines.
    • `nowrap` (default): All items stay on one line, potentially overflowing.
    • `wrap`: Items wrap to the next line when they run out of space.
    • `wrap-reverse`: Items wrap in reverse order.
  • `flex-flow` shorthand: Combines `flex-direction` and `flex-wrap`.

Using `flex-wrap: wrap;` is crucial for creating responsive layouts where items stack or flow differently on smaller screens.


.responsive-container {
    display: flex;
    flex-wrap: wrap; /* Allows items to wrap to next line */
    justify-content: space-around;
}
.responsive-item {
    flex-basis: 300px; /* Default width */
    flex-grow: 1; /* Allows items to grow to fill space */
    margin: 10px;
}

Day 18: Mastering Element Positioning!

The `position` Property

The `position` property controls how an element is positioned on the page. Combined with `top`, `right`, `bottom`, and `left`, it offers precise control.

  • `position: static;` (Default): Elements are positioned according to the normal flow of the document. `top`, `right`, `bottom`, `left` have no effect.
  • `position: relative;`: Positions the element relative to its normal position. `top`, `right`, `bottom`, `left` values will shift it from where it *would* normally be. Other elements are not affected by this shift.
  • `position: absolute;`: Positions the element relative to its *nearest positioned ancestor* (an ancestor with `position` other than `static`). If no positioned ancestor, it's relative to the `<html>` element. It's removed from the normal document flow.
  • `position: fixed;`: Positions the element relative to the viewport. It stays in the same place even when the page is scrolled. Useful for sticky headers/footers.
  • `position: sticky;`: A mix of `relative` and `fixed`. Behaves relatively until a certain scroll point, then becomes fixed.

Day 18: Positioning Properties: `top`, `right`, `bottom`, `left`, `z-index`

Controlling Location and Stacking Order

  • `top`, `right`, `bottom`, `left`: These properties are used *with* `position` (except `static`) to specify the element's offset from its positioning context. 
    
.box {
    position: relative;
    top: 20px; /* Moves 20px down from its normal top */
    left: 10px; /* Moves 10px right from its normal left */
}
  • `z-index`: Controls the stacking order of positioned elements. Elements with a higher `z-index` value appear on top of elements with a lower value. Only works on positioned elements (`relative`, `absolute`, `fixed`, `sticky`). 
    
.element-on-top {
    position: absolute;
    z-index: 10; /* Will appear above elements with z-index less than 10 */
}

Day 19: Enhancing Visuals with Backgrounds & Borders!

Images, Gradients, and Rounded Corners

CSS offers powerful ways to style backgrounds and borders, adding depth and visual interest.

  • `background-image`: Sets an image as the background.

  • `background-repeat`: Controls if/how a background image repeats (`no-repeat`, `repeat-x`, `repeat-y`, `repeat`).

  • Give all your buttons (e.g., submit button on contact form) `border-radius` and a `box-shadow`.
  • Consider adding a `border-radius` to images in your gallery.
  • Use `rgba()` colors for backgrounds or borders to add a touch of transparency.

These details make a huge difference in how professional your site looks!
